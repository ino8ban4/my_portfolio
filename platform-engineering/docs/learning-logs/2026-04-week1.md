## Issue #11: ArgoCD（GitOps）

### やったこと
- ArgoCDをminikubeにインストール
- Applicationマニフェストを作成しexpress-api-chartをArgoCD管理下に置いた
- ConfigMap/Secretテンプレートの欠落を発見・修正しGitOpsサイクルを完成させた

### 理解の証明

#### 理解内容
- ArgoCDはGitをSingle Source of Truthとして扱うPull型のCDツール
- デフォルトはポーリング（3分）でwebhookで即時化できる
- ApplicationリソースでGitのパスとk8sのnamespaceを紐付ける

#### 判断基準
- prune: true → Gitから消えたリソースはk8sからも消す
- selfHeal: true → 手動変更があってもGitの状態に戻す
- automated なし → 手動SYNC（本番環境向け）

#### 実装での選択理由
- ConfigMap/SecretをHelmテンプレートに含めた理由：ArgoCDはGitにないリソースを作れないため、全リソースをGit管理下に置く必要がある

- CreateContainerConfigError → ConfigMap未作成が原因、テンプレート追加で解決
