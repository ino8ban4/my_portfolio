# 2026-04 Week2 学習ログ

## Issue #12: CI/CDフルサイクル構築（GitHub Actions + ArgoCD）

### やったこと
- GitHub ActionsのCI完成（ビルド → ghcr.ioにpush → values.yaml自動更新）
- ArgoCDとの連携によりCI/CDフルサイクルを実現
- マルチアーキテクチャビルド（amd64/arm64）対応

### 理解の証明

#### 理解内容
- GitHub Actions（CI）: ビルド・テスト・パッケージ化を自動化
- ArgoCD（CD）: Gitの変更を検知してk8sへ自動デプロイ
- 両者は役割が明確に分離されている

#### 判断基準
- ArgoCDのトリガーはGitへのcommitのみ（ghcr.ioのpushでは動かない）
- values.yamlのtag更新がArgoCDへの「デプロイ指示」になる
- マルチアーキテクチャ対応はQEMU + Buildxで実現

#### 実装での選択理由
- platforms: linux/amd64,linux/arm64 を追加した理由：GitHub ActionsはAMD64で動作するが、Mac（Colima）のminikubeはARM64のためexec format errorが発生したため

### トラブルシュート
- exec format error → GitHub ActionsのビルドがAMD64のみだったため、マルチアーキテクチャビルドに変更して解決
